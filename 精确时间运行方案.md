# 精确时间运行方案

## 问题
GitHub Actions的schedule不会在精确时间运行，会在cron时间点后0-15分钟内随机执行。

## 解决方案：使用 cron-job.org 外部触发

### 步骤1：生成GitHub Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 设置：
   - Note: `bilibili-monitor-trigger`
   - Expiration: 选择过期时间（建议90天或自定义）
   - Scopes: 勾选 `repo` 和 `workflow`
4. 点击 "Generate token"
5. **复制并保存token**（只显示一次！）

### 步骤2：注册 cron-job.org

1. 访问 https://cron-job.org/
2. 注册免费账户
3. 验证邮箱

### 步骤3：创建cron任务

#### 任务1：股市里的猩猩（9:25-9:45每分钟）

在 cron-job.org 创建多个任务：

**9:25-9:45 每分钟：**
- Title: `股市里的猩猩-9:25`
- URL: `https://api.github.com/repos/Alan757/bilibili-monitor/actions/workflows/monitor-xingu.yml/dispatches`
- Method: `POST`
- Headers:
  ```
  Authorization: token YOUR_GITHUB_TOKEN
  Accept: application/vnd.github.v3+json
  Content-Type: application/json
  ```
- Body: `{"ref":"main"}`
- Schedule: `Cron expression: * 1-2 25-44 * * 1-5` (UTC时间)
  - 对应UTC+8: 9:25-9:45

**9:45-10:00 每2分钟：**
- Title: `股市里的猩猩-9:47`
- Schedule: `Cron expression: 47,49,51,53,55,57,59 1-2 * * 1-5` (UTC时间)

**10:00-15:00 每30分钟：**
- Title: `股市里的猩猩-10:00`
- Schedule: `Cron expression: 0,30 2-6 * * 1-5` (UTC时间)

#### 任务2：鳄鱼派（9:30-15:00每30分钟）

- Title: `鳄鱼派-9:30`
- URL: `https://api.github.com/repos/Alan757/bilibili-monitor/actions/workflows/monitor-eyu.yml/dispatches`
- Schedule: `Cron expression: 0,30 1-6 * * 1-5` (UTC时间)

### 步骤4：时区转换参考

| UTC+8 时间 | UTC 时间 | cron-job.org 表达式 |
|-----------|---------|-------------------|
| 9:25-9:45 | 1:25-1:45 | `25-45 1 * * 1-5` |
| 9:47,49,51,53,55,57,59 | 1:47,49,51,53,55,57,59 | `47,49,51,53,55,57,59 1-2 * * 1-5` |
| 10:00-15:00 | 2:00-7:00 | `0,30 2-6 * * 1-5` |

### 步骤5：禁用GitHub Actions的schedule（可选）

如果使用外部触发，可以禁用GitHub Actions的schedule：

在 workflow 文件中注释掉 schedule：
```yaml
on:
  # schedule:
  #   - cron: '...'
  workflow_dispatch:
```

### 注意事项

1. **Token安全**：不要泄露GitHub Token，它有仓库访问权限
2. **Token过期**：定期更新token（建议90天）
3. **免费额度**：cron-job.org免费版足够使用
4. **时区**：cron-job.org使用UTC时间，注意转换

### 测试

配置完成后，可以在cron-job.org手动触发一次测试，看GitHub Actions是否能正常运行。
